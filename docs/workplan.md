# Roadmap Planner 작업 계획

이 문서는 `RoadmapPlanner SRS.pdf`를 백엔드 구현 관점에서 정리한 작업
계획입니다. SRS는 KAIST 전산학부 학생을 위한 모바일 학업 계획 앱을
정의하며, 핵심 모듈은 Authentication, Course Prerequisite Tree, Semester
Roadmap, Credit and GPA Calculator, Sidebar, Settings입니다.

## 1. 제품 목표

- KAIST 전산학부 학생이 졸업 진행 상황, 선수 과목 순서, 학기별 수강
  부담을 쉽게 이해하도록 돕습니다.
- 한국어와 영어 UI를 지원하며, 기본 언어는 한국어로 둡니다.
- 공식 커리큘럼 데이터와 졸업 요건을 정확하게 유지하고, 전체 클라이언트
  재배포 없이 갱신할 수 있게 합니다.
- Tree, Roadmap, Calculator, Sidebar, Settings에서 같은 사용자 상태가
  일관되게 반영되도록 동기화합니다.
- 캐시된 계획 데이터는 오프라인에서 읽기 전용으로 접근할 수 있게 하고,
  인증과 데이터 변경에는 네트워크 연결을 요구합니다.

## 2. 핵심 요구사항 그룹

### 인증

관련 SRS ID: `REQ-AUTH-001`부터 `REQ-AUTH-006`까지

- KAIST SSO 로그인, 로딩 또는 스플래시 화면, 성공 후 Tree 화면 이동, SSO
  오류 처리와 재시도, 세션 유지, 로그아웃을 제공합니다.
- TBD: SSO 프로토콜을 SAML 2.0으로 할지 OpenID Connect로 할지 확인해야
  합니다.
- SSO를 사용할 수 없는 경우에만 공식 KAIST 이메일 기반 예외 인증을 허용할
  수 있습니다.

### Course Prerequisite Tree

관련 SRS ID: `REQ-TREE-001`부터 `REQ-TREE-009`까지

- General Required 과목과 전공 분야 노드를 기준으로 접고 펼칠 수 있는 선수
  과목 계층 구조를 제공합니다.
- 과목 유형, 수준, 카테고리, 세부 분야, 학점, 설명을 포함합니다.
- 검색과 필터, 이수 과목 표시, 다음 수강 가능 과목 강조, minor 모드,
  Graduation Research, 비표준 인정 학점용 Custom placeholder를 지원합니다.
- 백엔드는 노드와 엣지 메타데이터를 포함한 클라이언트 친화적 그래프 응답을
  제공해야 합니다.

### Semester Roadmap

관련 SRS ID: `REQ-MAP-001`부터 `REQ-MAP-012`까지

- 스크롤 가능한 학기 그리드, 기본 8학기 로드맵, 추가 학기, 드래그 가능한
  과목 카드 배치, 현재 학기 표시, 학기별 학점 합계를 저장합니다.
- 과목이 선수 과목보다 앞선 학기에 배치되면 검증 결과와 경고 정보를
  제공합니다.
- 로드맵의 Planned 과목 학점은 Calculator의 예정 학점 합계에 동기화되어야
  합니다.
- 데이터베이스에서 사용자의 Planned 과목을 불러와 로드맵에 표시합니다.

### Credit and GPA Calculator

관련 SRS ID: `REQ-CALC-001`부터 `REQ-CALC-010`까지

- 카테고리별 이수 학점과 필수 학점을 계산합니다.
- KAIST 4.3 만점 기준으로 누적 GPA와 카테고리별 GPA를 계산합니다.
- 카테고리 필터, Not Taken 기본 상태, 성적 입력, 현재 학기 예정 학점,
  Graduation Research 진행률, Custom 과목을 지원합니다.
- Roadmap과 동기화하여 Planned 과목이 Calculator 데이터에 자동 반영되게
  합니다.

### Sidebar

관련 SRS ID: `REQ-SBAR-001`부터 `REQ-SBAR-007`까지

- 사이드바 펼치기와 닫기, 카테고리 필터, 키워드 검색, 선택한 과목 상세
  정보, Find in Tree 동작, Add to Planned Subjects 동작을 지원합니다.
- 백엔드는 검색과 과목 상세 데이터를 제공하고, 화면 이동과 하이라이트는
  클라이언트 책임으로 둡니다.

### Settings

관련 SRS ID: `REQ-SET-001`부터 `REQ-SET-005`까지

- 소리, 진동, 학업 트랙, 현재 트랙 표시와 수정을 저장합니다.
- 학업 트랙 선택은 Tree, Roadmap, Calculator의 요건 표시를 바꿔야 합니다.
- SRS 본문에는 언어와 테마 설정도 설명되어 있으므로, 공식 요구사항 표가
  소리, 진동, 학업 트랙 중심이더라도 설정 모델에는 언어와 테마를 포함합니다.

## 3. 백엔드 마일스톤

### 마일스톤 1: 과목정보 DB 설계 및 DB 구성

- 과목 코드, 과목명, 학점, 설명, 카테고리, 세부 분야를 저장합니다.
- 선수 과목 관계와 Tree 구성을 위한 관계 데이터를 저장합니다.
- 중복 과목 코드, 누락 학점, 깨진 선수 관계를 검증합니다.

### 마일스톤 2: Tree용 API 개발

- 과목정보 DB를 기반으로 선수 과목 Tree 데이터를 제공합니다.
- 카테고리와 세부 분야별 Tree 조회를 지원합니다.
- Tree 노드에 표시할 과목 메타데이터를 포함합니다.

### 마일스톤 3: Sidebar용 API 개발

- 과목 검색과 카테고리 필터를 제공합니다.
- 선택한 과목의 상세 정보를 제공합니다.
- Tree에서 특정 과목을 찾을 수 있도록 식별 정보를 제공합니다.

### 마일스톤 4: User 정보 DB 설계 및 DB 구성

- 사용자 기본 정보와 로그인 식별자를 저장합니다.
- 사용자별 과목 상태, 성적, 로드맵 배치를 저장합니다.
- Custom 과목과 인정 학점 정보를 저장할 수 있게 합니다.

### 마일스톤 5: 개발용 이메일 기반 로그인 API 개발

- KAIST 이메일 기반의 임시 로그인 흐름을 제공합니다.
- 로그인한 사용자를 기준으로 User 정보를 조회하거나 생성합니다.
- 정식 KAIST SSO 도입 전 개발과 테스트에 사용할 수 있게 합니다.

### 마일스톤 6: Semester Roadmap API 개발

- 사용자별 기본 8학기 로드맵을 제공합니다.
- 과목 배치, 이동, 삭제, 추가 학기 생성을 저장합니다.
- 현재 학기 선택과 학기별 학점 합계를 제공합니다.

### 마일스톤 7: Credit and GPA Calculator API 개발

- 사용자별 과목 상태와 성적을 기반으로 GPA를 계산합니다.
- 카테고리별 이수 학점과 예정 학점을 계산합니다.
- Graduation Research와 Custom 과목을 계산에 포함합니다.

### 마일스톤 8: 졸업요건 DB 설계 및 DB 구성

- 입학 연도별 졸업요건 기준값을 저장합니다.
- 전공, 부전공, 복수전공별 요건을 확장 가능하게 구성합니다.
- 카테고리별 필수 학점과 졸업연구 기준을 저장합니다.

### 마일스톤 9: 졸업요건 관련 API 개발

- 사용자 트랙과 입학 연도에 맞는 졸업요건을 제공합니다.
- Calculator에서 사용할 필요 학점 기준을 제공합니다.
- 졸업요건 기준 데이터의 변경을 API 응답에 반영합니다.

### 마일스톤 10: Settings API 개발

- 언어, 테마, 소리, 진동 설정을 저장합니다.
- 전공, 부전공, 복수전공 선택을 저장합니다.
- 졸업요건 API와 연결할 academic track 값을 관리합니다.

### 마일스톤 11: 로그인 인증 관련 기능 구현

- KAIST SSO 기반 정식 인증 흐름을 구현합니다.
- 세션 유지, 만료, 로그아웃을 구현합니다.
- 사용자별 데이터 접근을 인증 정보 기준으로 제한합니다.

### 마일스톤 12: API 인증과 보안 처리

- 상태 변경 API에 인증과 권한 검사를 적용합니다.
- 토큰, 세션, 전송 구간 보안을 점검합니다.
- 사용자 데이터 격리와 주요 계산 로직을 테스트합니다.

### 마일스톤 13: 내보내기와 품질 보강

- 로드맵과 과목 선택을 JSON 또는 CSV로 내보냅니다.
- 한국어와 영어 라벨 누락 여부를 점검합니다.
- SRS의 성능 목표를 기준으로 병목 가능성을 확인합니다.
